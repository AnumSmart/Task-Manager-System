# Analytics Service

Микросервис сбора и агрегации аналитических данных. Слушает события из Kafka, сохраняет метрики, формирует отчёты по задачам и загрузке сотрудников.

## Технологии

- **Go 1.22**
- **Kafka** — приём событий
- **PostgreSQL** — хранение агрегированных метрик
- **TimescaleDB** (опционально) — для эффективного хранения временных рядов
- **gRPC** — API для запроса отчётов

### Kafka топики (consumer)

Топик --------------------------------------------------- Событие --------------------------------------------------- Что собираем
tasks.created ---------------------------------------TaskCreatedEvent ------------------------------------ Количество созданных задач, по часам
tasks.completed ----------------------------------- TaskCompletedEvent --------------------------------- Время выполнения, количество завершённых
tasks.status.changed ---------------------------- TaskStatusChangedEvent -------------------------------------- Переходы между статусами
tasks.overdue -------------------------------------- TaskOverdueEvent -------------------------------------------- Количество просрочек
tasks.assigned ------------------------------------- TaskAssignedEvent ------------------------------------------- Загрузка сотрудников

### Процесс сбора аналитики

┌─────────────────────────────────────────────────────────────────┐
│ 1. Kafka consumer получает событие │
│ event := TaskCompletedEvent{TaskID, AssigneeID, CompletedAt} │
└─────────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────────┐
│ 2. Сохраняем сырое событие в analytics_events │
│ (для возможности пересчёта) │
└─────────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────────┐
│ 3. Агрегатор обновляет метрики: │
│ - task_metrics (daily) │
│ - user_metrics (daily) │
│ - time_series (почасовые данные) │
└─────────────────────────────────────────────────────────────────┘

### Метрики для дашборда (примеры)

Метрика ------------------------------------------ Описание ------------------------------------------ Формула
Всего задач ----------------------------- Количество созданных задач -------------------------- COUNT(tasks_created)
Выполняемость ---------------------------- Процент выполненных задач ------------------------ completed / created \* 100
Просрочки ------------------------------ Количество просроченных задач --------------------------- COUNT(overdue)
Среднее время выполнения ----------- Среднее время от создания до выполнения ------------------ AVG(completion_time)
Загрузка сотрудников ------------------- Количество задач на сотрудника ----------------------- tasks_assigned / users
Топ исполнителей -------------- Сотрудники с наибольшим количеством выполненных задач ------ ORDER BY tasks_completed DESC

### Graceful Shutdown

Сервис корректно завершает работу:

Останавливает consumer (не принимает новые сообщения)

Дожидается обработки текущих событий

Сохраняет накопленные метрики (если есть буфер)

Закрывает соединения с БД и Kafka

Завершает процесс
