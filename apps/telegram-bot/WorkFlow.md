┌─────────────────────────────────────────────────────────────────────────────┐
│ ЭТАП 2: Привязка бота │

└─────────────────────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ Руководитель открывает Telegram, находит бота, пишет /start │
│ │
│ Бот: "Добро пожаловать! Введите ваш email для привязки аккаунта" │
│ Руководитель: "admin@romashka.ru" │
│ │
│ Бот проверяет email в user-service: │
│ ├── Если найден → сохраняет telegram_chat_id = chat_id │
│ └── Если не найден → "Пользователь с таким email не найден" │
│ │
│ Бот: "✅ Аккаунт привязан! Теперь вы можете создавать задачи." │
└─────────────────────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────────────────────┐

└─────────────────────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ Руководитель создаёт задачу: /create → выбирает исполнителя из списка │
│ │
│ Система: │
│ 1. Создаёт задачу в task-service │
│ 2. Публикует событие в Kafka: task.assigned │
│ 3. Notification-service ловит событие, находит telegram_chat_id │
│ исполнителя и отправляет уведомление │
│ │
│ Сотрудник получает: "📋 Вам назначена задача: Подготовить отчёт" │
